Xiao Yun2025-07-302025-08-06

文件传输协议（FTP, File Transfer Protocol）是用于在服务器和客户端之间通信和传输文件的标准协议。FTP采用客户端-服务器架构。

一、FTP协议概念

FTP的基本概念

定义

FTP是File Transfer Protocol的缩写，中文名称文件传输协议。

作用

用于在网络上进行文件上传、下载和其他的文件操作。

工作模式

FTP采用客户端/服务器模式，用户通过客户端软件链接到服务器。

传输层协议

FTP使用TCP作为其传输协议，保证数据传输的可靠性和顺序性。

FTP的工作原理

  • FTP建立两个TCP连接：控制连接和数据连接
  • 控制连接用于传输命令和响应，保持客户端和服务器之间的会话，默认端口号：21
  • 数据连接用于传输实际的文件数据，可以根据需要建立和关闭，默认端口号：20
  • FTP支持主动模式（PORT）和被动模式（PASV）两种数据传输模式

FTP的两种数据传输模式

  • 主动模式（PORT）

    客户端向服务器发送端口信息，服务器主动连接客户端的端口。

  • 被动模式（PASV）

    服务器开启并发送端口信息给客户端，客户端连接服务器的端口。

FTP的主要功能

  • 上传文件：将本地计算机的文件上传到服务器。
  • 下载文件：将服务器上的文件下载到本地计算机。
  • 文件和目录管理：在服务器进行创建、删除、重命名等操作。
  • 支持多种数据传输模式：ASCII模式、二进制模式

FTP的优缺点

  • 优点

    简单易用，传输效率高，支持多种文件类型，广泛应用于各种场景。

  • 缺点

    传统的FTP协议不加密，安全性较低，容易受到中间人攻击。

FTP的替代方案

  • 为了提高安全性，可以使用SFTP（SSH File Transfer Protocol）或FTPS（FTP Secure）
  • SFTP使用SSH协议加密，提供更高的安全性。
  • FTPS使用TLS/SSL加密，提供更高的安全性。

FTP服务器

  • FTP服务器是运行FTP协议的服务器软件，用于提供文件共享和传输服务。
  • FTP服务器可以支持匿名访问，也可以要求用户进行身份验证。
  • FTP服务器可以根据需要进行配置，以限制用户对特定文件的访问。

总结：FTP是一种重要的文件传输协议，在网络中广泛应用虽然存在安全性方面的考虑，但可以使用安全的替代方案，仍然可以进行安全的文件传输。

二、在Ubuntu22.04上配置FTP

FTP在Ubuntu上可用，但需要一个FTP守护程序来提供安全隧道来加密FTP服务器中数据流。VSFTPD（非常安全的FTP守护进程）是Linux系统上可靠的FTP服务器。

2.2、创建登录用户

创建登录用户

我们将创建user1、user2，以及匿名用户，并且使ftp服务器实现以下的功能：

  • user1用户作为本地最高级用户在ftp中几乎不受影响，可以访问全部目录，可以上传、可以下载、可以删除、新建目录、新建文件。
  • user2用户作为本地的用户，ftp登录后限制在自己的主目录内(/home/user2)，在其主目录下可以上传、可以下载、可以删除、新建目录、新建文件。
  • 匿名用户（其他的任何人，不需要专门的密码就可以访问）限制在/home/ftp目录下，且只有下载权限。
