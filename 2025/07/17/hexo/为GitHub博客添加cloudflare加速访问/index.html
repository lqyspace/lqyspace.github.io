为GitHub博客添加cloudflare加速访问

Xiao Yun2025-07-172025-07-17

1、前言

本人在前段时间在github上搭建了一个个人博客，由于在国内直接访问github速度会很慢或者直接不能访问，因此寻思着使用CDN(content delivery network)将网页内容分发到全球各地的服务器上，同时还能缩短网站的加载时间。

但是大部分国内的CDN服务都是收费的，所以把目光转向cloudflare。cloudflare作为全球最大的网络服务提供商，提供了免费的cdn服务，虽然cdn的节点在国外，但是总比直接访问github要快得多，不过免费版请求次数限制10万次，不过对于博客而言已经绰绰有余了，下面介绍一下配置过程。

2、准备

首先需要给自己配置一个域名，本人在 dynadot官网 上购买的域名，购买教程就跳过了。

3、教程

3.1、配置cloudflare

1、进入cloudflare ，注册账号并登录。

2、在首页中点击 账户主页，并且点击 添加域。

3、然后输入你购买的域名，注意不要带www或者https， 下面的类型选择推荐的既可。

4、接下来网站会提示你选择套餐，选择free即可。

5、点击继续后cloudflare会自动扫描域名的dns记录，如果是刚刚建好的域名，可能扫描的结果为空，截图中的几条记录可以不用管。

3.2、修改DNS服务器

1、进入我们的域名管理后台，修改DNS的配置。

2、点击DNS设置。

3、配置即可。

过一段时间DNS就可以生效了。

返回cloudflare，如果看到”Cloudflare正在保护您的站点”，说明配置成功。

3.3、设置Github Page

进入github.io对应的仓库，进入setting，点击左栏的Pages，在 Custom domain 中输入自己的域名，点击 save，如果成功会显示下图：

这样我们就可以通过域名来访问我们的博客啦，还可以在上图中勾选 Enforcrs HTTPS，这样网站仅会通过https提供服务。

info

如果如下图显示dns配置不正确，推测可能是使用了cloudflare后，GitHub验证DNS时返回的是cdn服务器的ip地址，而不是在cloudflare上开始配置的4个GitHub page的ip地址，可以在线dig一下自己的域名验证一下。不过只要网站能通过域名正常访问就没什么问题。
